# CartPole + 调库 DQN / PPO 实验报告

## 1. 实验目的

- 在 CartPole 这种连续状态环境中，理解为什么不能直接建 Q 表。
- 结合调库训练过程，理解 DQN/PPO 这类方法为何要用深度网络做函数近似。
- 记录 `gamma` 和 `learning rate` 对 reward 曲线、收敛速度和稳定性的影响。

## 2. 实验环境与任务

- 环境：`CartPole-v1`
- 任务：控制小车左右移动，使杆尽可能长时间不倒下
- 算法：调库实现 DQN（可扩展对比 PPO）

## 3. CartPole 的 MDP 结构

### 3.1 状态空间 `S`

每个时刻的状态由 4 个连续变量组成：

1. 小车位置（cart position）
2. 小车速度（cart velocity）
3. 杆角度（pole angle）
4. 杆角速度（pole angular velocity）

这 4 维状态是连续值，因此无法像离散网格世界那样直接建立有限 Q 表。

### 3.2 动作空间 `A`

动作是离散的 2 个：

- 向左施加力
- 向右施加力

注意：动作不是“移动多少”，而是施加固定方向的力，位移由动力学系统自然演化得到。

### 3.3 状态转移 `P(s'|s,a)`

`P` 不是简单写成 `1`。更准确地说：

- 在 MDP 定义中，`P(s'|s,a)` 表示从状态 `s` 执行动作 `a` 到达下一状态 `s'` 的概率分布。
- 对 CartPole，可理解为由物理动力学主导的状态转移（在给定状态和动作下基本确定，但在 MDP 形式中仍写作转移概率）。

我的理解是：虽然代码里看起来“执行动作就得到下一个状态”，但在 MDP 公式里仍然用概率转移去描述它。

### 3.4 奖励函数 `R`

- 每执行一步且未终止，奖励 `+1`
- 回合终止时结束累计

因此回合总回报本质上就是“存活步数（存活时间）的度量”。

### 3.5 折扣因子 `gamma`

`gamma` 控制未来回报在当前决策中的权重，是时间视野的核心参数。

## 4. 关键现象与分析

### 4.1 Reward 本质等于存活时间

在 CartPole 中，单步奖励规则几乎固定为“活着就 +1”，所以平均回报提升可直接解释为策略让杆保持平衡的时间更长。

也就是说，这个任务里 reward 的提升，基本就是“每局能坚持更久”。

### 4.2 `gamma` 如何改变时间视野

DQN 的目标项可写为：

```text
Q(s,a) <- r + gamma * max_a' Q(s',a')
```

- `gamma` 较小：更关注短期奖励，学习速度可能更快，但更容易短视。
- `gamma` 较大：更重视长期回报，通常更稳定、更可能达到更高最终表现，但收敛更慢。

原因是 `gamma^k` 会随步数增长而衰减，`gamma` 越小，远期奖励被压缩得越快。

这和我的观察一致：`gamma` 小时曲线有时起得快，但后期稳定性和最终表现不一定最好。

### 4.3 `learning rate` 对收敛与稳定性的影响

- `lr` 大：参数更新幅度大，对误差反应强，早期学习快，但震荡明显，不稳定风险高。
- `lr` 小：参数更新平滑，训练更稳，但收敛速度慢。

因此 `lr` 本质上是在“学习速度”和“训练稳定性”之间做权衡。

按我的理解，`lr` 可以看成模型“纠错力度”：纠错太猛会抖，纠错太小会慢。

### 4.4 为什么训练峰值不等于真实能力

训练曲线中的短暂高峰不一定代表模型已经真正学会稳定策略，原因包括：

- 探索噪声（如 epsilon-greedy）可能偶然走出高分轨迹；
- 单回合结果方差较大，存在随机波动；
- 训练阶段策略和评估阶段策略不完全一致。

所以应关注“多回合平均表现”和“稳定性”，而不是单次峰值。

## 5. 结论

1. CartPole 的连续状态空间使 Q 表法不可扩展，必须依赖函数近似（深度网络）。
2. 该任务的回报实质是生存时间，reward curve 可以直观反映策略能力变化。
3. `gamma` 决定时间视野，`lr` 决定更新强度；二者共同影响收敛速度与稳定性。
4. 评估模型能力应以稳定平均表现为主，不能仅看训练过程中的瞬时峰值。
