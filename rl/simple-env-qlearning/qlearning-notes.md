## 2. 环境与奖励设计

环境是 8 个状态的分支图：

- 安全分支：`START -> SAFE_1 -> SAFE_2 -> SAFE_3 -> GOAL`
- 风险分支：`START -> RISK_1 -> RISK_2 -> (GOAL 或 TRAP)`

奖励规则：

- 到达 `GOAL`：`+5`（终止）
- 到达 `TRAP`：`-5`（终止）
- 到达 `RISK_1 / RISK_2`：`+1`（不终止，短期诱惑）

并且在 `RISK_2` 执行向右动作时加入滑落概率（`RISK_SLIP_PROB=0.35`），让风险分支具有随机性。

## 3. 实验设置

- `gamma ∈ {0.1, 0.3, 0.5, 0.7, 0.9}`
- `alpha ∈ {0.1, 0.3, 0.5, 0.7, 0.9}`
- `epsilon = 0.2`
- 每组参数用 5 个随机种子训练并取均值
- 评估指标：`avg_return`、`success_rate`

## 4. 结果观察（结合趋势图）

- 低 `gamma`（如 0.1、0.3）整体表现较差，说明策略偏短视，更容易被风险分支的即时奖励吸引。
- 中高 `gamma`（0.7、0.9）在若干 `alpha` 下表现更好，成功率可接近或达到 1.0，说明长期回报建模更充分。
- `alpha` 不是越大越好：中等学习率通常更稳，过大时会出现波动。
- 曲线存在抖动是正常现象：环境有随机性（滑落），且训练使用探索策略。

## 5. 理论问题回答

### 5.1 为什么 Bellman 可以递归更新？

因为“当前状态-动作价值”可以分解为：

- 当前一步即时奖励 `r`
- 下一状态的最优未来价值 `max Q(s', a')`

即：

```text
Q(s,a) <- Q(s,a) + alpha * [r + gamma * max_a'Q(s',a') - Q(s,a)]
```

这个关系是“当前值 = 一步奖励 + 折扣后的未来值”，所以可以不断用新采样递归逼近真实价值。

### 5.2 为什么 gamma < 1 有助于收敛？

- `gamma < 1` 会让更远未来的回报按 `gamma^k` 衰减，避免无限累计导致值函数发散。
- 折扣后，Bellman 更新更稳定，数值更容易收敛到固定点。
- 直观上，`gamma` 越小，时间视野越短；`gamma` 越大，时间视野越长但学习更难稳。

### 5.3 为什么 exploration 必须存在？

如果只有贪心（不探索），初期 Q 值几乎都一样，策略会被早期偶然结果锁死，只走一条路径，可能永远发现不了更优策略。  
epsilon-greedy 用小概率随机动作，保证有机会访问新状态并修正错误估计。

### 5.4 为什么负奖励会向前传播？

在 TD 更新里，上一状态会使用下一状态的价值：

- 若某状态后面经常通向 `TRAP`（负回报），它的 `Q` 会先变小；
- 再通过 `target = r + gamma * max Q(s')` 传递到更前面的状态；
- 最终在更早决策点也体现为“这条路不划算”。

所以负奖励不是只影响当前一步，而是会沿着状态转移链条向前回传。

## 6. 本次结论

- 这个分支环境能直观看到“短期收益 vs 长期风险”的权衡。
- `gamma` 主要影响时间视野，`alpha` 主要影响更新快慢与稳定性。
- 探索、折扣和 TD 递推三者共同决定了 Q-learning 的学习效果。
